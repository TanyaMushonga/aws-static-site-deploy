Complete Guide to Deploy Static Websites using S3 and CloudFront

This guide walks you through deploying a static website on Amazon Web Services (AWS) using S3 for hosting and CloudFront as a Content Delivery Network (CDN). This setup provides global content distribution, improved performance, and SSL/TLS encryption.

üìö Step-by-Step Deployment Process

1Create and Configure S3 Bucket

First, create an S3 bucket to store your static website files:

# Create S3 bucket (bucket names must be globally unique) aws s3 mb s3://your-unique-bucket-name # Enable static website hosting aws s3 website s3://your-unique-bucket-name \ --index-document index.html \ --error-document error.html
‚ö†Ô∏è Important: Bucket names must be globally unique across all AWS accounts. Choose a descriptive, unique name.

2Configure Bucket Policy for Public Access

Set up bucket policy to allow public read access:

{ "Version": "2012-10-17", "Statement": [ { "Sid": "PublicReadGetObject", "Effect": "Allow", "Principal": "*", "Action": "s3:GetObject", "Resource": "arn:aws:s3:::your-unique-bucket-name/*" } ] }

Apply the policy:

aws s3api put-bucket-policy \ --bucket your-unique-bucket-name \ --policy file://bucket-policy.json

3Upload Website Files

Upload your static website files to the S3 bucket:

# Upload all files recursively aws s3 sync ./your-website-folder/ s3://your-unique-bucket-name/ # Set proper content types for better performance aws s3 cp s3://your-unique-bucket-name/ s3://your-unique-bucket-name/ \ --recursive \ --metadata-directive REPLACE \ --content-type "text/html" \ --exclude "*" \ --include "*.html"

4Create CloudFront Distribution

Set up CloudFront distribution for global content delivery:

# Create distribution configuration file cat > cloudfront-config.json << EOF { "CallerReference": "$(date +%s)", "Comment": "Static website distribution", "DefaultCacheBehavior": { "TargetOriginId": "S3-your-unique-bucket-name", "ViewerProtocolPolicy": "redirect-to-https", "MinTTL": 0, "ForwardedValues": { "QueryString": false, "Cookies": {"Forward": "none"} } }, "Origins": { "Quantity": 1, "Items": [ { "Id": "S3-your-unique-bucket-name", "DomainName": "your-unique-bucket-name.s3.amazonaws.com", "S3OriginConfig": { "OriginAccessIdentity": "" } } ] }, "Enabled": true, "DefaultRootObject": "index.html" } EOF # Create the distribution aws cloudfront create-distribution --distribution-config file://cloudfront-config.json

5Configure Custom Error Pages

Set up custom error pages for better user experience:

# Update distribution to include custom error pages aws cloudfront update-distribution \ --id YOUR_DISTRIBUTION_ID \ --distribution-config '{ "CustomErrorResponses": { "Quantity": 2, "Items": [ { "ErrorCode": 404, "ResponsePagePath": "/error.html", "ResponseCode": "404", "ErrorCachingMinTTL": 300 }, { "ErrorCode": 403, "ResponsePagePath": "/error.html", "ResponseCode": "404", "ErrorCachingMinTTL": 300 } ] } }'

6Set Up Custom Domain (Optional)

Configure a custom domain for your website:

  1. Request SSL certificate in AWS Certificate Manager (ACM)
  2. Add alternate domain names (CNAMEs) to CloudFront distribution
  3. Update DNS records to point to CloudFront distribution
# Request SSL certificate aws acm request-certificate \ --domain-name yourdomain.com \ --domain-name www.yourdomain.com \ --validation-method DNS

üí° Troubleshooting Common Issues

  • 403 Forbidden: Check bucket policy and public access settings
  • 404 Not Found: Verify index document configuration and file paths
  • Slow Loading: Check CloudFront cache settings and origin configuration
  • SSL Issues: Ensure certificate is validated and properly configured
  • DNS Problems: Verify CNAME records point to correct CloudFront domain

üìä Monitoring and Maintenance

Set Up CloudWatch Monitoring

# Create CloudWatch alarm for 4xx errors aws cloudwatch put-metric-alarm \ --alarm-name "CloudFront-4xx-Errors" \ --alarm-description "Monitor 4xx errors" \ --metric-name 4xxErrorRate \ --namespace AWS/CloudFront \ --statistic Average \ --period 300 \ --threshold 5 \ --comparison-operator GreaterThanThreshold

üéâ Deployment Complete!

Your static website is now deployed and accessible via CloudFront. The distribution domain will be available in the AWS Console, and your site will be globally distributed for optimal performance.
